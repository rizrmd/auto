## Error Boundary Behavior Reference

### What Error Boundaries CATCH ‚úÖ

| Error Type | Caught? | Example |
|------------|---------|---------|
| Render errors | ‚úÖ YES | `throw new Error()` during render |
| Lifecycle method errors | ‚úÖ YES | Error in `componentDidMount()` |
| Constructor errors | ‚úÖ YES | Error in `constructor()` |

### What Error Boundaries DO NOT CATCH ‚ùå

| Error Type | Caught? | Solution |
|------------|---------|----------|
| Event handlers | ‚ùå NO | Use try/catch in handler |
| Async code | ‚ùå NO | Use try/catch with async/await |
| Server-side rendering | ‚ùå NO | Handle on server |
| Errors in Error Boundary itself | ‚ùå NO | Use parent Error Boundary |

**Example - Proper Error Handling:**
```typescript
// ‚ùå NOT caught by Error Boundary
const handleClick = () => {
  throw new Error('Event error');
};

// ‚úÖ Proper handling
const handleClick = () => {
  try {
    // risky operation
  } catch (error) {
    console.error('Error:', error);
    // show user feedback
  }
};
```
